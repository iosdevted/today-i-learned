## 🚀 Intro

1. 나를 한 문장으로 함축적으로 요약하는 것은 중요합니다
→ 나의 방향성과 미래를 결정짓는 신념

2. 프레젠테이션(면접)은 정말 나의 이야기 하는 자리. 진정성있게

3. 면접이나 인터뷰 할 때 타인의 시선을 신경쓰지 않고 당당하게 이야기 할 수 있는 태도
→ 코딩만 잘 해서 될 것이 아니라, 코딩을 잘 하는 "나"를 셀링할 수 있는 능력을 갖춰야 함

4. 설득의 3요소

Ethos (발표자): 에너지, 카리스마, 태도
Logos (이성)
Pathos (감성)

## 🙋‍♀️ Ethos

1. 우리는 왜 스피치를 하는 걸까?

- `나의 생각과 아이디어를 다른 사람에게 전달하는 것`
- 내 생각이 흔들리지 않아야, 다른 사람에게 신뢰를 얻고 받아들일 수 있도록 마음을 열 수 있다

2. 회사의 비전과 나의 비전을 일치시켜야 합니다

- `"나의 능력을 이용해 회사의 OO 발전에 기여하고 OO한 능력을 펼치고 싶습니다"`
  - 라고 이야기 할 수 있어야 함
- 기업규모에 따른 인터뷰 포인트
스타트업 → 나의 성장 가능성
  - 대기업 → 성장가능성, 태도와 로열티 등...

3. "오늘 이 말은 꼭 하고 올 거야!"

- `오늘 나에 대해 어필하고 싶은 부분 3가지에 대해 미리 정하고` 면접장으로 향하기
- 어떤 질문이 나오든지 내가 하고 싶은 말과 연결 시키는 것은 중요합니다
  - 그렇지 않으면, 면접관의 질문에 대한 수동적인 대답만 하게되고
    - 미련이 남을 수 있음
- 그럼 나의 강점 3가지는 뭘까...? 나중에 적어봐야 겠다

4. 유대인의 전통 대화 교육법 "하브루타"

    1. `연장자에게 발언권을 준다`

    2. `이야기 도중에 끼어들지 않는다`

    3. `말하기 전에 먼저 생각한다`

    4. `당황하여 서둘러 대답하지 않는다`

        - 바꿔 생각해보면 면접은 구직 기간동안 한 번 뿐인 기회이기 때문에

        - 너무 위축되지 말고 즐기는 것이 좋겠다

        - 돈 주고도 하기 어려운 경험이기 떄문에, 나도 이 기회를 최대한 활용하자

        - 생각을 바꿀 수 있는 너무 좋은 얘기였습니다

    5. `질문과 대답을 간결하게 한다`

    6. `처음 할 이야기와 나중 할 이야기를 구별하여 한다`

        - 내 부족함을 드러내는 질문을 마지막에 받았다면?

        - 내가 말하고 싶은 3가지 또는 신념으로 나에 대한 이미지를 정리 하고 나와야 한다

        - 긍정적으로 마무리 하는게 좋다 (조금 철판 깔더라도)

    7. 잘 알지 못하고 말했거나 잘못 말한 것은 솔직하게 인정한다

        - 어차피 아는지 모르는지는 다 알고 있음
        - 다음에 더 준비해서 대답 하겠다는 약속을 하는 것은 좋음

5. 나의 바디 랭귀지

- 당연히 자세도 중요함
- 허리 어깨 가슴 펴고
- 손은 제스쳐를 하고 (어차피 손은 할 게 없다. 제스쳐가 되려 임팩트 있음)

## 🙆‍♀️ Logos

1. 좋은 스피치의 기본은 청중 분석

- 면접관은 누구인지
- 나만의 차별점은 무엇일까
- 면접관은 왜 지금 사람을 뽑고 있는지
- 어떤 프로젝트를 입사 후에 수행해야 하는가
- 이 청중에서의 키맨은 누구인가
- ✨ 결국 본질은 같다. 함께 일할 사람을 뽑는 것 ✨

2. 현업 개발자분들께 전해 들은 실제 채용 과정 예시

    1. 서류 전형

        - 어떤 언어를 쓰고, 어떤 프로젝트를 했는지
        - 인의 기여도를 항상 잘 기술해야 함

    2. 기술 면접

        1. 개발 책임자가 면접을 진행 ( + 실무자 )
        2. 프로젝트에서 내가 기여한 부분에 대한 정확한 검증을 함
            - 나는 어떤걸 개발했는가
            - 개발 하면서 무엇이 어려웠는가
                - 어떻게 해결했는가
        3. 실무 스킬 검증
            - 면접 중의 화이트보드 코딩, 손 코딩 결과에 대해 질의 응답
                - 미리 손 코딩은 연습을 해보는게 좋다 (의외로 잘 안됨)
            - 또는 사전 과제, 사전 프로젝트 결과에 대해 질의 응답
            - 지금 바로 투입되면 얼마나 퍼포먼스를 낼 수 있는지
        4. 협업 능력 검증
            - 실제 질문 사례
                - 어느 팀에서 어떤 포지션으로 일했는지
                - 오픈소스는 어떤 걸 사용해 보았는지
                - 기존 익숙한 업무와 전혀 다른 팀에 배치가 되면 어떻게 할 것인지
                - 여태까지 어떻게 공부했는지
                - 코드리뷰를 해봤다면, 어떻게 진행했는지
                - 개발 프로세스에 대한 이해가 있는지
    3. 임원 면접 / 대표 면접

        - 컬쳐 핏 (우리 회사랑 잘 맞나?)
            - 꽤 중요하게 생각하는 포인트
                - 그리고 회사에 대한 Royalty 👑 체크
                - 왜냐면 능력 있으면 언제든 떠날 수 있으니까
            - 인성과 Attitude

    4. 스타트업의 경우

        - 지속적으로 발전 가능한 인재인지
        - 기획과 디자인에 대한 경험이나 이해
        - 다양한 롤을 수행할 수 있는지

3. 마지막에는 나를 기억에 남도록 하는 하나의 메시지

- 인용구나 내가 지어낸 말이나 아무거나 관계없다
- 유명한 개발자의 말을 빌려서라도 인상에 남는 이야기를 하는 것도 좋음
- 나의 인생이라는 긴 글에 제목을 붙여주는 중요한 포인트가 됨

## 🙇‍♀️ Pathos

1. 🙋‍♀️ 좋은 발표란 뭘까요?

→ 나만의 언어로, 생생하게 살아있는 말로 전달하는 것

→ 스크립트가 있다면, 나 만의 언어가 아닌 "글 말"이 된다. 이런 발표가 진정성이나 감동이 있을까?

2. 🔥 열정은 조심히 전달해야 한다

- 중함을 갖추고 진심을 다해

- 자칫 무례해 질 수 있음

3. 🌊 발표 자료는 전체 Flow 를 머릿속에 담아 주자

- 그래서 청중을 이끌고 발표를 할 수 있다

4. 🕋 압박면접을 이겨내는 법

- 쿠션언어로 극복하자

  - "역시 예리하시군요"

  - "중요한 말씀을 해주셨네요"

  - "충분히 그렇게 생각하실 수 있는 일입니다"

  - "중요한 부분을 짚어 주셔서 감사합니다"

- 쿠션 언어로 질문자의 불편한 마음을 누그러뜨리자
